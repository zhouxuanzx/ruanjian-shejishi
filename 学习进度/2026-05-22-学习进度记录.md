## 二、上午选择题 · 六模块速记表

### 🔷 模块1：软件工程（12-15分）

```
生命周期模型：
  瀑布 → 需求明确变更少     V模型 → 测试驱动
  增量 → 分批交付           螺旋 → 风险驱动
  敏捷Scrum → 三角色(产品负责人/Master/团队)+四会议

软件测试：
  单元测试→白盒(看代码)    集成测试→灰盒(测接口)
  系统测试→黑盒(看功能)    回归测试→改后重测

维护类型（改适完预）：
  改正性→改bug(20%)  适应性→改环境(25%)
  完善性→加功能(50%★最高)  预防性→防未来(5%)

内聚(低→高)：偶逻时过通信顺序功能
耦合(高→低)：内公外控标数据
核心原则：高内聚、低耦合
```

### 🔷 模块2：数据库系统（10-12分）

```
ER图三要素：
  实体(矩形)  属性(椭圆)  联系(菱形)
  联系类型：1:1  1:n  m:n

关系代数（四种运算）：
  σ(选择)→选行    π(投影)→选列
  ×(笛卡尔积)    ⋈(自然连接)

范式：
  1NF→属性不可再分
  2NF→消除部分依赖（主键单属性自动满足）
  3NF→消除传递依赖
  BCNF→消除主属性依赖

SQL基础：SELECT 列 FROM 表 WHERE 条件
          GROUP BY 分组  ORDER BY 排序
事务ACID：原子性 一致性 隔离性 持久性
```

### 🔷 模块3：操作系统（8-10分）

```
进程状态转换：
  就绪→运行(调度)  运行→阻塞(等待事件)
  运行→就绪(时间片到)  阻塞→就绪(事件完成)

PV操作：
  S>0 → 可用资源数    S<0 → 等待进程数（绝对值）

死锁四条件：互斥 请求保持 不可剥夺 环路等待
银行家算法 → 找安全序列（只记结论）

存储管理：
  页式存储：逻辑地址 = 页号 + 页内地址
  存储器速度：寄存器 > Cache > 主存 > 外存
```

### 🔷 模块4：数据结构与算法（8-10分）

```
线性表：
  顺序表→随机访问快，插入删除慢
  链表→随机访问慢，插入删除快
  栈→后进先出(LIFO)    队列→先进先出(FIFO)

二叉树遍历：
  前序：根左右    中序：左根右
  后序：左右根    层次：从上到下

排序算法（时间+稳定性）★重点：
  O(n²)：冒泡√  插入√  选择✗
  O(nlogn)：快速✗  归并√  堆✗
  √=稳定  ✗=不稳定

查找：
  顺序查找 O(n)    二分查找 O(logn) (要求有序)
```

### 🔷 模块5：计算机组成（6-8分）

```
CPU组成：运算器  控制器  寄存器
Cache作用：解决CPU与主存速度不匹配

存储器层次（速度从高到低）：
  寄存器 → Cache → 主存(内存) → 外存(硬盘)
  容量递增，速度递减

总线分类：
  数据总线(传数据)  地址总线(传地址)  控制总线(传控制信号)

CISC vs RISC：
  CISC→复杂指令集，指令多，长短不一
  RISC→精简指令集，指令少，长度固定
校验码：奇偶校验(检错) CRC(检错+纠错)
```

### 🔷 模块6：计算机网络 + 法规（8-11分）

```
OSI七层(上→下)：应表会传网数物
TCP/IP四层(上→下)：应用层 传输层 网际层 网络接口层
传输层→端到端可靠传输（TCP）

TCP(面向连接,可靠,三次握手) vs UDP(无连接,快,不可靠)
端口：HTTP:80  HTTPS:443  FTP:21  DNS:53

IP分类：
  A类:1~126   掩码255.0.0.0   网络位8
  B类:128~191 掩码255.255.0.0 网络位16
  C类:192~223 掩码255.255.255.0 网络位24

IP地址二进制判断：
  A类→第一个字节最高位0
  B类→第一个字节最高两位10
  C类→第一个字节最高三位110

法规保护期限：
  著作权→生+50  发明→20年  实用/外观→15年
  商标→10年(可续)  商业秘密→无限
```

## 四、下午案例题 · 万能答题模板

### 第1题：数据流图 DFD（15分）

```
答题四步法：
  第一步：找外部实体 → 系统外的人/系统（矩形框）
  第二步：找数据存储 → 以"文件""表""库"结尾（平行线）
  第三步：找加工 → 动词+名词（圆形）
  第四步：找数据流 → 名词短语（箭头）

四大常见错误：
  黑洞 → 加工只有输入没有输出
  奇迹 → 加工只有输出没有输入
  父图子图不平衡 → 顶层图与0层图数据流不一致
  直接穿越 → 数据流不经加工直连实体和存储
```

### 第2题：数据库设计 ER图（15分）

```
答题四步法：
  第一步：识别实体 → 题目中的核心名词
  第二步：识别联系及类型 → 1:1  1:n  m:n
  第三步：识别属性 → 主键用下划线标出
  第四步：ER图转关系模式 ★核心：
    1:1联系 → 合并到任意一方
    1:n联系 → 合并到n方
    m:n联系 → 单独生成表，含双方主键+联系属性

符号速记：实体(矩形)  属性(椭圆)  联系(菱形)
```

### 第3题：UML建模（15分）

```
只考两种图：用例图 + 类图

用例图：
  参与者(小人) ── 用例(椭圆)
  包含<<include>>：必须执行，虚线箭头
  扩展<<extend>>：可选执行，虚线箭头
  泛化：空心三角箭头，继承关系

类图：
  类(矩形分三层)：类名 | 属性 | 方法
  关联：普通实线
  泛化：空心三角（继承）
  聚合：空心菱形 ◇（整体-部分，可分离，如球队→球员）
  组合：实心菱形 ◆（整体-部分，不可分离，如订单→订单项）
```

### 第4题（算法题）+ 第5题（面向对象题）

```
策略：保5-8分，放弃最后一问

算法题：
  第一问→填算法名称（看代码特征）
  第二问→填时间复杂度（背表格）
  第三问→填空缺（i++、if(a>b)等基础语句）
  第四问→放弃

面向对象题：
  第一问→填类名/继承关系
  第二问→填方法名
  第三问→填空缺（this.name=name）
  第四问→放弃
```
