### 🔴 新增模块1：程序设计语言与编译原理（6分）

```
一、编译与解释的区别
  编译器：整体翻译成目标代码后执行（如C）
  解释器：逐条翻译执行，不产生目标代码（如Python）
  编译后执行速度快，解释后跨平台性好

二、编译过程6阶段（顺序必考）：
  词法分析 → 语法分析 → 语义分析 → 中间代码生成 → 代码优化 → 目标代码生成
  口诀：词法语中间优目

三、正规式（词法分析工具）：
  | → 或（a|b = a或者b）
  * → 0次或多次（a* = ε,a,aa,aaa...）
  + → 1次或多次（a+ = a,aa,aaa...）

  真题示例：
  正规式(0|1)*101 描述的语言：所有以101结尾的01串

四、有限自动机：
  DFA：每个状态对每个字符只有一个转移（确定的）
  NFA：每个状态对每个字符可有多个转移（不确定的）

  解题方法：从初态出发，逐个字符按箭头走，看能否停留在终态
  
五、文法分类（乔姆斯基）：
  0型→无限制文法
  1型→上下文有关文法（αAβ → αγβ）
  2型→上下文无关文法（A → γ，产生式左部单个非终结符）
  3型→正规文法（A → aB 或 A → a）
  
  口诀：0无限 1有关 2无关 3正规
        程序设计语言用2型（上下文无关文法）

六、中缀转后缀（逆波兰式）：
  中缀：a + b × c
  后缀：a b c × +
  
  方法：操作数直接输出，运算符按优先级入栈
       优先级：() > */ > +- 
       同优先级：从左往右
```

### 🟡 新增模块3：多媒体基础（1-3分）

```
一、图像相关
  分辨率：图像像素点数（如1920×1080）
  颜色深度：每个像素占的位数（如24位真彩）
  图像容量(B) = 分辨率 × 颜色深度 / 8

二、音频相关
  采样频率：每秒采样次数（如44.1kHz）
  采样精度：每次采样的位数（如16位）
  声道数：单声道/立体声
  音频容量(B) = 采样频率 × 采样精度 × 声道数 × 时长(s) / 8

三、视频相关
  视频容量 = 单帧图像容量 × 帧率 × 时长

四、常见标准
  JPEG → 静态图像压缩（有损）
  MPEG → 视频压缩
  MP3 → 音频压缩
  GIF → 动图，256色

五、媒体分类
  感觉媒体、表示媒体、显示媒体、存储媒体、传输媒体
```

### 🔴 新增模块4：流水线技术（计算机组成·新增重点）

```
一、流水线周期 = 各段执行时间中最长的那个

二、流水线执行时间：
  k段流水线，各段时间分别为Δt1,Δt2,...,Δtk
  周期 = max(Δt1,Δt2,...,Δtk)
  执行n条指令时间 = (k + n - 1) × 周期

三、吞吐率 TP = n / 流水线执行时间
   最大吞吐率 = 1 / 周期

四、加速比 S = 串行执行时间 / 流水线执行时间

真题示例：
流水线5段，各段时间2,3,2,3,2ns，执行10条指令。
  周期 = 3ns
  流水线时间 = (5+10-1)×3 = 42ns
  串行时间 = (2+3+2+3+2)×10 = 120ns
  吞吐率 = 10/42 ≈ 0.24GIPS
  加速比 = 120/42 ≈ 2.86
```

### 🔴 新增模块5：数据表示（计算机组成·新增重点）

```
一、四种码制：
  正数：原码=反码=补码（最高位0）
  负数：
    原码→符号位1，数值位不变
    反码→原码符号位不变，数值位取反
    补码→反码+1
    移码→补码符号位取反

  补码作用：将减法变加法，统一处理符号位

二、浮点数表示：
  N = M × 2^E
  M为尾数（定点小数）  E为阶码（定点整数）
  阶码决定范围，尾数决定精度

三、进制转换：
  二→十：按权展开求和
  十→二：整数除2取余，小数乘2取整
  二→八：3位一组
  二→十六：4位一组
```

### 🟡 新增模块6：进程调度 + 页面置换（操作系统·补充）

```
一、进程调度算法：
  FCFS  → 先来先服务（非抢占）
  SJF   → 短作业优先（平均等待时间最短）
  优先级 → 按优先级调度（可能饥饿）
  时间片轮转 → 每个进程轮流执行固定时间片

二、页面置换算法（虚拟内存）：
  FIFO → 淘汰最先调入的页（Belady异常）
  LRU  → 淘汰最近最久未使用的页（★最优常用）
  OPT  → 淘汰将来最远使用的页（理论最优，无法实现）
  Clock → 近似LRU，环形扫描

三、缺页率 = 缺页次数 / 总访问次数

四、I/O控制方式（从低到高）：
  程序查询 → CPU轮询，效率最低
  中断方式 → 外设主动通知CPU
  DMA    → 直接内存访问，无需CPU干预
  通道方式 → 专用I/O处理器
```

### 🟡 新增模块7：图论（数据结构·补充）

```
一、图的存储：
  邻接矩阵（n×n，适合稠密图）
  邻接表（链表，适合稀疏图）

二、图的遍历：
  DFS（深度优先，用栈）
  BFS（广度优先，用队列）

三、拓扑排序：
  → 无环有向图的线性排序
  → 判断方法：邻接矩阵为三角矩阵时必有拓扑排序
  → 每次选入度为0的结点输出

四、关键路径：
  → 从开始到结束的最长路径
  → 决定项目最短工期
  → 关键路径上的活动（关键活动）没有时间余量

五、最短路径：
  Dijkstra算法 → 单源最短路径（边权非负）
  Floyd算法  → 多源最短路径
```

### 🟡 新增模块8：面向对象设计原则（选择题+案例）

```
六大设计原则（SOLID+）：

1. 单一职责原则：一个类只负责一项职责
2. 开闭原则：对扩展开放，对修改关闭
3. 里氏替换原则：子类可以替换父类
4. 依赖倒置原则：依赖抽象，不依赖具体
5. 接口隔离原则：使用多个专门接口，不用单一总接口
6. 迪米特法则：最少知识原则，减少耦合

记忆：单开里倒接迪
```

## 二、下午案例题新增补充

### 🔴 新增模块9：UML其他图（序列图/活动图/状态图）

```
UML图分类：
  静态（结构图）：类图 对象图 组件图 部署图
  动态（行为图）：用例图 序列图 活动图 状态图 通信图

═══════════════════════════════════════

一、序列图（顺序图）—— 下午第3题常考！

元素：
  对象（顶部矩形）      生命线（竖虚线）
  激活期（竖长条矩形）   消息（横向箭头）

消息类型：
  同步消息 → 实心三角箭头（调用方法，等待返回）
  异步消息 → 开口箭头（发送信号，不等返回）
  返回消息 → 虚线箭头（返回值）

答题：根据文字描述，填对象名/消息名/消息顺序

═══════════════════════════════════════

二、活动图

元素：
  活动结点（圆角矩形）   起始结点（实心圆）
  终止结点（实心圆+环）  判断结点（菱形）
  分岔/汇合（粗横线）    泳道（分区线）

分岔(fork)：一个控制流分成多个并行流
汇合(join)：多个并行流合并成一个

泳道：用于区分不同角色/部门执行的活动

═══════════════════════════════════════

三、状态图

元素：
  状态（圆角矩形，上写状态名）
  转换（箭头，标注[事件]/[动作]）
  初态（实心圆）  终态（实心圆+环）

描述：单个对象在其生命周期内的状态变化
例：订单→[待支付→已支付→已发货→已完成]

═══════════════════════════════════════

四、通信图（协作图）

类似序列图，但侧重空间关系而非时间顺序
对象之间用直线连接，标注消息序号和名称
```
