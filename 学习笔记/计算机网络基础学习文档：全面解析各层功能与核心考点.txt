## 三、概念延伸与细节追问

### （一）TCP 协议相关

1. **SYN 和 ACK 标志位分别是什么含义？**

    - SYN：TCP 里的同步标志位，用来发起连接时同步双方的初始序列号，让后续数据传输能按顺序来，不乱套

    - ACK：确认标志位，用来告诉对方 “我收到你发的数据了”

2. **TCP 序列号是什么？为什么初始序列号不固定从 0 开始？**

    - 序列号作用：就像给每段数据标页码，初始序列号是起始页码，比如客户端从 100 开始发，第一个数据字节就是 100，服务器用 ACK=101 确认，说明收到 100 了，下次要 101，这样数据到了能按顺序拼，不会乱

    - 不固定从 0 的原因：TCP 会把大块数据切成小段发，初始序号不固定是为了避开旧连接的残留切片，防止混进来乱了新数据

3. **TCP 三次握手和四次挥手的完整流程是什么？**

    - 三次握手（建立连接）：客户端先发 SYN 同步包，带上自己的初始序列号；服务器收到后，回 SYN\+ACK，既同步它的序列号，也确认客户端的；客户端再发 ACK 确认服务器的同步，连接就建好了。其中服务器回的 ACK 号是客户端 SYN 的初始序列号加 1，客户端第三次发的 ACK，也是服务器 SYN 的序列号加 1

    - 四次挥手（断开连接）：客户端发 FIN，服务器先回 ACK，等服务器数据发完再发 FIN，客户端回 ACK 才彻底断

### （二）IP 地址分类相关

1. **A/B/C 类 IP 地址的网络位、主机位分别是多少？地址范围是多少？**

    - A 类：8 位网络位，首位固定 0，网络范围 \[1\.0\.0\.0\]\(1\.0\.0\.0\) 到 \[126\.0\.0\.0\]\(126\.0\.0\.0\)，主机位 24 位，能连 1600 多万台主机

    - B 类：16 位网络位，前两位固定 10，范围 \[128\.0\.0\.0\]\(128\.0\.0\.0\) 到 \[191\.255\.0\.0\]\(191\.255\.0\.0\)，主机位 16 位，撑 6 万多台

    - C 类：24 位网络位，前三位固定 110，范围 \[192\.0\.0\.0\]\(192\.0\.0\.0\) 到 \[223\.255\.255\.0\]\(223\.255\.255\.0\)，主机位 8 位，最多 254 台

2. **A/B/C 类 IP 地址的范围是怎么计算出来的？**

    - A 类：首位固定 0，剩下 7 位网络号，最小 0000001 是 1，最大 01111110 是 126，所以 1\-126 开头，127 留作环回地址单独用

    - B 类：前两位 10，后面 14 位网络号，最小 10000000 是 128，最大 10111111 是 191，所以 128\-191 开头

    - C 类：前三位 110，后面 21 位，最小 11000000 是 192，最大 11011111 是 223，所以 192\-223 开头

3. **IP 地址的十进制数是什么意思？**
IP 地址本身的四个分段就是十进制数，比如 A 类的 \[10\.0\.0\.1\]\(10\.0\.0\.1\)，第一个 “10” 就是 1\-126 这个十进制范围里的数，对应二进制开头是 0。

### （三）OSI 七层网络模型相关

1. **OSI 七层模型的层级顺序是什么？各层对应什么实例？**

    - 从下到上顺序：物理层、数据链路层、网络层、传输层、会话层、表示层、应用层

    - 对应实例：物理层比如网线传电信号；数据链路层交换机用 MAC 地址传帧；网络层路由器靠 IP 转发数据包；传输层 TCP 保证网页数据完整到；会话层远程桌面维持连接；表示层 HTTPS 加密数据；应用层浏览器用 HTTP 拿网页

2. **数据在 OSI 七层模型中的完整传输流程是什么？**
用浏览器打开网页：应用层 HTTP 生成请求内容；表示层把内容加密；会话层和服务器建会话；传输层 TCP 把请求切成段标序号；网络层 IP 加地址找路径；数据链路层打包成帧通过交换机传；物理层网线把信号送到服务器，再反向一层层解回来，就能看到网页了。

### （四）网络层与数据链路层相关

**网络层和数据链路层的功能有什么区别？IP 地址和 MAC 地址分别起什么作用？**

- 网络层：靠 IP 地址做 “跨网导航”，比如在上海发数据去北京，IP 先标好北京的网络地址，路由器按这个地址选路，把数据从上海的网络转到北京的网络。IP 是跨网络的全局地址

- 数据链路层：到了目标局域网，把网络层的 IP 包封装成 “帧”，加上本地设备的 MAC 地址，交换机按 MAC 直接把帧发给目标设备。MAC 地址是设备在本地网络的 “身份证号”

- 简单类比：IP 管 “从这城到那城”，MAC 帧管 “到了这楼送哪户”
