## 一、编译与解释的区别（必考1题）

| | 编译 | 解释 |
|------|------|------|
| 原理 | 整体翻译成目标代码后执行 | 逐条翻译执行 |
| 产物 | 生成目标代码（.exe/.class） | 不产生目标代码 |
| 速度 | 执行快 | 执行慢 |
| 代表 | C、C++ | Python、JavaScript |

> Java是先编译（java→class）再解释（JVM），两者都用。

---

## 二、编译过程6阶段（必考顺序题）

```
词法分析 → 语法分析 → 语义分析 → 中间代码生成 → 代码优化 → 目标代码生成
```

口诀：**词法语中间优目**

| 阶段 | 做什么 |
|------|--------|
| 词法分析 | 扫描字符流，识别单词符号（标识符、关键字、常数） |
| 语法分析 | 根据语法规则识别语法单位（表达式、语句），生成语法树 |
| 语义分析 | 检查语义错误（类型匹配等） |
| 中间代码生成 | 生成与机器无关的中间表示 |
| 代码优化 | 改进中间代码提高效率 |
| 目标代码生成 | 生成机器代码或汇编代码 |

## 三、正规式（词法分析工具）

- `|` → 或（a|b = a或者b）
- `*` → 0次或多次（a* = ε, a, aa, aaa...）
- `+` → 1次或多次（a+ = a, aa, aaa...）

真题示例：`(0|1)*101` 描述的语言 → **所有以101结尾的01串**

## 四、有限自动机

| | DFA（确定） | NFA（不确定） |
|------|------|------|
| 转移 | 每个状态对每个字符唯一转移 | 每个状态对每个字符可有多个转移 |
| 空转移 | 不允许 | 允许 |

解题方法：从初态出发，逐个字符按箭头走，看能否停在终态。

## 五、文法分类（乔姆斯基）

| 类型 | 名称 | 规则 | 应用 |
|------|------|------|------|
| 0型 | 无限制文法 | 最宽泛 | — |
| 1型 | 上下文有关 | αAβ → αγβ | — |
| 2型 | 上下文无关 | A → γ（左部单个非终结符） | **程序设计语言** |
| 3型 | 正规文法 | A → aB 或 A → a | 词法分析 |

口诀：**0无限 1有关 2无关 3正规**

---

## 六、中缀转后缀（逆波兰式）

规则：
1. 操作数直接输出
2. 运算符按优先级入栈
3. 优先级：`()` > `*/` > `+-`
4. 同级从左往右
5. 遇到 `)` 弹栈直到 `(`

例：`a + b × c` → `a b c × +`

## 实战真题

1. 编译过程中，将源代码的字符流转换为单词符号的阶段是（ ）
A. 语法分析 B. 词法分析 C. 语义分析 D. 代码优化

2. 以下属于解释型语言的是（ ）
A. C B. C++ C. Python D. Java

3. 正规式 `a(b|c)*` 可以表示的字符串是（ ）
A. a B. abc C. acb D. ba

4. 乔姆斯基文法分类中，程序设计语言通常采用（ ）
A. 0型文法 B. 1型文法 C. 2型文法 D. 3型文法

5. 中缀表达式 `(a+b)*c-d` 的后缀形式为（ ）
A. ab+c*d- B. ab+cd*- C. a+b*c-d D. abc*+d-

---

## 真题答案

1. **B** — 词法分析负责识别单词符号
2. **C** — Python是解释型，C/C++是编译型，Java先编译后解释
3. **A** — `a(b|c)*`：以a开头，后面跟0个或多个b或c。A=`a`（0个b/c），B有a、b但不匹配规则ab后c，C同理，D以b开头不匹配
4. **C** — 2型（上下文无关文法）用于描述程序设计语言语法
5. **A** — `ab+c*d-`：`(a+b)`→`ab+`，`×c`→`ab+c*`，`-d`→`ab+c*d-`
